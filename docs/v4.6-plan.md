> 状态:计划已锁定方向,待开实现 agent。
> 决策(用户 2026-06-10 拍板):

## 0. 一句话目标

发布 **4.6**:在 **UI/UX 层全面超越 Tolaria**(用真正的设计系统 + Vue 组件库取代散装手写 CSS),并补齐与 Tolaria 的 **7 项功能差距**(属性检查器、类型驱动侧栏、类型化关系、关系图谱、保存的过滤视图、Inbox 工作流、tldraw 白板)。

**验收标准(Definition of Done):**
1. 全应用统一在一套设计 token + 组件库之上;不再有 56 个 `<style scoped>` 各写各的弹窗/下拉/按钮。
2. 7 项功能全部可用,且每项都通过 `solomd-dev` MCP bridge 端到端自测 + Rust 单测(涉及 Rust 时)+ CLI/MCP 冒烟。
3. 数据全部 Markdown-first:新功能只读写 YAML frontmatter / fenced block / `.solomd/` sidecar,vault 与 Tolaria 交叉兼容。
4. 14 语言字符串补齐;明暗 + 8 主题不回归。
5. 与 Tolaria vault 双向打开互不破坏(交叉兼容测试)。

## 1. 现状诊断 —— 为什么我们“看起来原始”

来自前端外壳测绘(`app/src` 全量扫描):

| 维度 | 现状 | 问题 |
|---|---|---|
| 设计系统化程度 | ~40% | 有 `--bg/--text/--accent` 等 11 个核心色 + 8 主题,但**没有 spacing/radius/shadow/z-index 比例尺** |
| 组件库 | 几乎没有 | 按钮/下拉/弹窗/输入框都是**内联 markup + scoped CSS**,每个容器各写一份 |
| 重复 | 严重 | `.dropdown__menu` 复制 5 次、模态 backdrop+圆角+阴影+z-index 复制 5+ 次 |
| 样式孤岛 | 56 个 | 56 个组件各自 `<style scoped>`,阴影/边框/密度互不一致 |
| 硬编码 | 多处 | `rgba(255,159,64,0.08)` 等魔法值内联,而非 `--accent-soft` |

**根因**:没有“token 层 + headless 组件库”这一中间层。Tolaria 之所以高级 = Tailwind 4 的 `@theme` token 纪律 + Radix/cva 组件原语 + 一致密度/focus ring/微交互。**我们补的就是这一层。**

✅ 已有可复用的好东西:`Icons.vue`(集中 SVG)、`Toast.vue` + toasts store、`themes.ts` 主题管线、CodeMirror 主题生成器 —— 这些保留。

## 2. 设计系统(4.6 的地基 / 全员共享契约)

> 这是“全并行”里唯一的 trunk:所有 7 个功能 agent **从这份书面契约**取色取组件,而不是各自发明。即便并行,这份 spec 先写定,避免返工。

### 2.1 Token 层 —— `app/src/styles/tokens.css`(新)

单一 `:root` token 源,明暗 + 8 主题只覆盖语义层。比例尺(此前完全缺失):

```
/* 间距(8pt 基)*/      --sp-1:4px --sp-2:8px --sp-3:12px --sp-4:16px --sp-5:24px --sp-6:32px
/* 圆角 */              --r-sm:4px --r-md:8px --r-lg:12px --r-full:999px   (主基调 8px)
/* 阴影(克制)*/        --sh-1: 0 1px 2px rgba(0,0,0,.06)
                        --sh-2: 0 4px 12px rgba(0,0,0,.08)
                        --sh-pop: 0 8px 28px rgba(0,0,0,.12)
/* 边框 */              --bd:1px solid var(--border)
/* z 层 */              --z-pane:10 --z-sticky:50 --z-pop:1000 --z-modal:2000 --z-toast:3000
/* 动效 */              --ease:cubic-bezier(.2,.7,.2,1) --dur-fast:120ms --dur:180ms
/* 焦点环 */            --ring: 0 0 0 2px var(--bg), 0 0 0 4px var(--accent)
```

色彩语义层(沿用现有变量名,补齐缺口;**强调色锁定暖橙**):
```
--accent:#ff9f40  --accent-fg:#1a1a1a  --accent-soft:rgba(255,159,64,.10)  --accent-ring:rgba(255,159,64,.45)
--bg --bg-elev --bg-hover --bg-active --border --text --text-muted --text-faint --danger --success --warning
```
> 具体明暗数值见 §2.4(由 Tolaria 设计调研提取后对齐 SoloMD 暖色基调)。

### 2.2 Vue Headless 组件库 —— `app/src/ui/`(新)

用 Vue 实现(不引 Tailwind/Radix,纯 token + 少量组合式逻辑),12 个原语,全应用替换内联实现:

| 组件 | 取代现有 | 备注 |
|---|---|---|
| `DsButton` | 各处 `button`/`.icon-btn` | variant: primary/ghost/subtle/danger;size sm/md |
| `DsInput` / `DsTextarea` | SettingsPanel 内联 input | 统一 focus ring |
| `DsSelect` / `DsDropdown` | 5 份 `.dropdown__menu` | 锚定+teleport 复用 `anchoredDropdown.ts` |
| `DsModal` / `DsDialog` | 5 份 backdrop | 统一 backdrop/圆角/阴影/Esc/焦点陷阱 |
| `DsPopover` | 无 | 日期/状态选择器用 |
| `DsCalendar` | 无 | 属性检查器 DateCell |
| `DsChip` / `DsBadge` | 标签/状态内联 | 状态色、tag 色 |
| `DsPanel` | `.rs-pane-host` | 侧栏面板统一头部/grip/关闭 |
| `DsListRow` | 各列表项 | hover/active/选中态一致 |
| `DsTooltip` | 无 | |
| `DsContextMenu` | `.sidebar-ctx` | 右键菜单 |
| `DsTabs` | TabBar/PaneTabBar 公共部分 | |

### 2.3 外壳重构

- `App.vue` 的三栏 + 右栏 6 面板 + 状态栏迁到 `DsPanel`/token;不改信息架构,只换底层。
- `CommandPalette`、`SettingsPanel`、`QuickSwitcher`、所有 Dialog 改用 `DsModal`。
- `SettingsPanel` 表单全部走 `DsInput/DsSelect/DsButton`,消除“每个设置一份 raw input”。
- 8 主题改为只覆盖语义 token(数值收敛)。

### 2.4 Tolaria 设计 token 提取结果(已对齐 SoloMD 暖橙)

Tolaria 实测数值(供我们对齐密度/层级/暖中性灰),**强调色一律换成 SoloMD 暖橙**:

| 角色 | Tolaria 明 | Tolaria 暗 | SoloMD 4.6 采用 |
|---|---|---|---|
| app 底 | `#FFFFFF` | `#1F1E1B` | 明 `#FFFFFF` / 暗 `#1F1E1B`(暖黑,非纯黑)|
| 侧栏底 | `#F7F6F3` | `#191814` | 同(暖灰,这是高级感关键)|
| 面板/卡 | `#FFFFFF` | `#23221F` | 同 |
| 按钮底 | `#EBEBEA` | `#34322D` | 同 |
| 边框 | `#E9E9E7` | `#34322D` | 同 |
| 正文 | `#37352F` | `#E6E1D8` | 同(暖深灰,非 `#000`)|
| 次要文字 | `#787774` | `#B8B1A6` | 同 |
| 弱文字 | `#B4B4B4` | `#7F776D` | 同 |
| **强调** | `#155DFF`(蓝)| `#78A4FF` | **`#ff9f40` 明 / `#ffa54d` 暗**(我们的橙,替换蓝)|
| **焦点环** | 蓝 | 蓝 | **橙 `#ff9f40`**(3px,这是“高级”的关键细节)|
| 语义 | green `#38A169` red `#E53E3E` | lift 版 | 沿用(非主色)|

字体:UI = `Inter, -apple-system, …`;等宽 = `IBM Plex Mono`;**base 14px / line-height 1.5 / `font-synthesis:none` / antialiased**。
圆角:base **8px**(sm 2 / md 6 / lg 8 / xl 12)。阴影:xs/sm/md/lg 渐进。过渡:**150–200ms**。

**“高级感”三要素(我们照做)**:① token 纪律(语义命名,不内联魔法值)② 一致密度(按钮 h-36px、gap 8px、卡 padding 24px)③ 统一 3px 焦点环 + 150ms 微过渡 + 暖中性灰(不用刺眼纯黑白)。我们只把蓝换成橙、把 Radix/cva 换成 Vue headless,其余照搬这套纪律。

## 3. 七项功能工作流

> 共性:**Markdown-first,绝大多数零新增 Tauri 命令**(复用 `workspace_index_*` + `write_file`/`read_file`)。重活=对着 §2 组件库写 Vue。每个功能=一个独立 worktree agent。

| # | 功能 | 数据模型(全部 Markdown-first) | 复用 | 新增 Tauri | 工时 | 依赖 |
|---|---|---|---|---|---|---|
| F1 | **属性检查器** ⌘⇧I | YAML frontmatter 为唯一真源;类型推断在渲染期(扩展 `lib/bases.ts`);类型覆盖/置顶存 `.solomd/properties.json` | workspaceIndex.frontmatter、bases 推断、wikilinks、BacklinksPanel 导航范式、右栏 pane 机制 | `update_frontmatter_property` / `delete_frontmatter_property`(serde_yaml 外科式改写,保 body 字节不变) | 6.5 | 设计系统 + 属性引擎 |
| F2 | **类型驱动侧栏** | `type:<Name>` 成员 + `type:Type` 定义笔记(标题=类型名,`_icon/_color/_order/_pinned_properties`);镜像 Tolaria,vault 交叉兼容 | workspaceIndex(type 已解析)、FileTree 侧栏范式、F1 的属性值格式化 | MVP 无(成员写走 `write_file`+js-yaml splice) | 5 | 设计系统 + 属性引擎 |
| F3 | **类型化关系** | 任意非保留 frontmatter 键含 `[[wikilink]]` 即为关系;正向写在 YAML,反向在内存跨库计算 | wikilinks/backlinks、workspaceIndex | `workspace_index_referenced_by`(服务端反向解析,复用内存索引) | 5 | workspaceIndex(已有) |
| F4 | **关系图谱(Neighborhood)** | 零新增约定;关系=值为 wikilink 的键;出边=这些键,入边=反向扫描 | workspace_index_files/backlinks/resolve | 无(v1 客户端;大库可选 `workspace_index_neighborhood` 下个 minor) | 3.5 | workspaceIndex |
| F5 | **保存的过滤视图** | 每视图一文件 `.solomd/views/{slug}.yml`,schema 复用并扩展 `bases.ts`(扁平 AND → 递归 all/any 树) | bases 引擎、workspaceIndex、侧栏 pane 机制、settings toggle 范式 | 无(`write_file`/`read_file`/`list_dir`);可选 MCP/CLI 动词下个 minor | 6 | 属性引擎(bases 泛化) |

| F7 | **tldraw 白板** | 保留 Tolaria 磁盘格式:fenced ` ```tldraw id=.. height=.. ` body 为 `TLStoreSnapshot` JSON;交叉兼容 | 文件读写、git 历史、backlinks | 无(快照内联;可选 `whiteboard_export_png` 下个 minor) | 6 | 设计系统 |

**合计 ~35.5 人天**(并行下墙钟取决于最慢链 = F1 6.5d)。

### 关键技术注记
- **F1 YAML 往返保真**是最大风险:serde_yaml 0.9 不保注释、可能重排键/重引号 → git 噪声 diff。**对策**:外科式 block 编辑(只改目标键、原位保序、body verbatim)+ `frontmatter_edit_test.rs` 表驱动 fixture(CJK 键、内联数组、引号、块标量 body、无 frontmatter 合成、前导 H1)。
- **F5 牵动 `bases.ts`**:`applyFilters` 从扁平 AND 升级为递归 `FilterGroup`,需兼容现有扁平调用方 → 这是“属性引擎”地基的一部分,F1/F2 也受益。
- **F7 tldraw 是重依赖**(~1-2MB)且在 Tauri/缩放下脆:需 zoom-aware viewport 补丁、文本测量守卫、平台权限拒绝守卫、dialog 重新宿主 —— 单独 worktree,最后合并,体积用动态 import 隔离。

## 4. Agent 分配 & 并行编排

**依赖图:**
```
        ┌─────────────────────────────────────────┐
   [地基] 设计系统(tokens+ui/ 组件库)  +  属性引擎(bases.ts 泛化)
        └───────────────┬─────────────────────────┘
                        ↓ 全员从书面契约 §2 + §F5 取用
   F1 属性  F2 类型侧栏  F3 关系  F4 图谱  F5 视图  F6 Inbox  F7 白板
        └─────────────── 统一合并 + 集成自测 ──────────┘
```

**编排策略(worktree 隔离,呼应“全并行最后合并”):**
1. **第 0 阶段(地基,~1 天,trunk)**:1-2 个 agent 落地 `tokens.css` + `ui/` 12 组件 + `bases.ts` 泛化,并把外壳/SettingsPanel/CommandPalette/Dialog 迁过去。**先合入 `feat/v4.6` 主干**,因为它是单一共享文件契约,7 个功能都 import 它——这是“全并行”里物理上无法绕过的唯一串行点(否则 7 份各发明颜色=必返工)。
2. **第 1 阶段(7 功能,全并行)**:7 个 agent,各自 `isolation: worktree`,从 `feat/v4.6` 切出,互不碰对方文件(功能间文件几乎零重叠——见 §3 new_files)。每 agent 自带自测。
3. **合并阶段**:按 F6→F4→F3→F2→F1→F5→F7 顺序回合并(轻→重、依赖少→多),每次合并后跑集成自测;冲突面主要在 `App.vue`(右栏 pane 注册)、`settings.ts`(toggle)、`CommandPalette.vue`、`i18n/en.ts` —— 这几个文件用**追加式注册**(每功能加自己的行)把冲突降到最小。

> 说明:用户选了“全并行”。我在**功能层**完全并行(7 个同时跑);仅把“设计系统契约”作为 trunk 先定,这是正确的工程解读,不是退回“UI 先行再等确认”。

## 5. 自测策略(每功能强制)

每个功能 agent **声明完成前必须**:
1. **磁盘往返**:`solomd_write_file` 造 fixture → 驱动功能 → `solomd_read_file` 断言 body 字节级正确、只改目标。
2. **UI 路径**:`solomd_dev_click`/`_text`/`_eval` 驱动真实 UI(需 `pnpm tauri dev`)+ `solomd_screenshot` 视觉核验。
3. **Rust 单测**(涉及 Rust 时):`cargo test`(F1 的 frontmatter 往返、F3 的反向解析)。
4. **Git 安全**:`solomd_git_status` 前后比对,确认单次编辑 = 干净最小 diff。
5. **CLI + MCP 冒烟**:按全局规则 #4,新增能力同步暴露到 mcp-server + CLI example(读优先;写 gate)。
6. **i18n**:新字符串进 `en.ts`,14 语言占位补齐。

> 合规红线:**自测未过不得声明完成**(全局 CLAUDE.md #2)。tldraw 这类合成键盘事件驱动不动的,退到 store 级 + 截图核验(见记忆 `reference_dev_bridge_synthetic_events`)。

## 7. 风险与回滚

- **全并行合并冲突**:用追加式注册 + worktree;`App.vue`/`settings.ts` 设“注册区”注释锚,降冲突。
- **设计回归**:M0 先在样板页 + 主界面截图基线,逐组件替换时 before/after 比对。
- **YAML 噪声 diff(F1)**:外科式编辑 + 往返 fixture 把关,未过不合。
- **tldraw 体积/脆性(F7)**:动态 import,失败优雅降级为只读 JSON 预览;单独最后合并,可单独回滚不影响其余 6 项。
- **范围蔓延**:SoloMD 无原生 Type 系统时,F1 砍掉 Tolaria 的 Type/Workspace 选择器;各功能“可选下个 minor 项”(MCP/CLI 动词、neighborhood/png 导出 Rust 命令)按记忆 `feedback_cli_mcp_cadence` 推迟,不塞进本版。
