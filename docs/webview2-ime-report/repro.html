WebView2 中文输入法 (CJK IME) bug — contentEditable vs <textarea>

How to reproduce:

  1. Open this page in a WebView2 host (or in Microsoft Edge for comparison).
  2. Switch to a Chinese IME — Sogou Pinyin (搜狗拼音) shows it most severely; Microsoft Pinyin may be milder.
  3. Click the contentEditable box and type a sentence with punctuation, e.g. nihao，shijie。 (pinyin + space to commit, then Chinese comma/period).
  4. Do the same in the textarea box.

Observed in contentEditable (BUG): the first character after focusing is dropped (“吃字”), and Chinese punctuation must be typed twice before it commits. The same input in the plain <textarea> works correctly.
